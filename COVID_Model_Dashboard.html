‚ñº üå≤ Model 1: Random Forest Regressor

Architecture
RandomForestRegressor( n_estimators=200, # 200 decision trees max_depth=10, # Max depth per tree min_samples_split=5, # Min samples to split node min_samples_leaf=2, # Min samples per leaf random_state=42, # Reproducibility n_jobs=-1 # Use all CPU cores )
How It Works
  1. Bootstrap Sampling: Randomly sample 80% of training data (with replacement) for each tree
  2. Feature Selection: At each node, randomly select subset of features
  3. Tree Growing: Split on feature that minimizes mean squared error (MSE)
  4. Aggregation: Final prediction = average of all 200 tree predictions
Hyperparameter Rationale
Parameter Value Reason
n_estimators 200 Balance between accuracy and computation time
max_depth 10 Prevent overfitting while capturing patterns
min_samples_split 5 Require sufficient data before splitting
min_samples_leaf 2 Avoid overly specific leaf nodes

‚ñº üöÄ Model 2: Gradient Boosting Regressor ‚≠ê BEST

Architecture
GradientBoostingRegressor( n_estimators=200, # 200 boosting stages max_depth=5, # Shallower trees (vs RF=10) learning_rate=0.05, # 5% contribution per tree subsample=0.8, # 80% subsample (stochastic) random_state=42, # Reproducibility loss='squared_error' # MSE loss function )
How It Works (Sequential Learning)
  1. Initialize: Start with mean prediction
  2. For each of 200 stages:
    • Compute residuals = actual - predicted
    • Train new tree to predict residuals
    • Add tree √ó learning_rate to ensemble
    • Update predictions
  3. Final Prediction: Sum of all tree contributions

üîë Key Idea: Each tree corrects previous errors, like an iterative refinement process

Why GB Beats Random Forest
  • ‚úÖ Residual Learning: Trees predict mistakes, not raw targets
  • ‚úÖ Sequential Boosting: Each tree builds on previous success
  • ‚úÖ Stochastic Updates: 80% subsample prevents overfitting
  • ‚úÖ Gentle Learning: 5% learning rate = stable convergence
  • ‚úÖ Shallow Trees: max_depth=5 generalizes better

‚ñº üéØ Key Findings

1Ô∏è‚É£ Health Metrics Drive Predictions

Mortality rate, ICU usage, and oxygen support account for 8.8% of predictive power beyond baseline Cases 24h feature.

Implication: Severity indicators are strong secondary predictors. During peaks, hospital metrics could serve as alternative signals when case numbers aren't immediately available.

2Ô∏è‚É£ Quarantine Measures Show Limited Predictive Power

Quarantine data only 0.9% importance (Quarantine + Quarantine_per_Case combined)

Why? Likely because:

  • Quarantine is reactive (follows cases, doesn't predict them)
  • Data may be incomplete or noisy
  • Effect occurs with lag (need 7-day, 14-day lookback)

Implication: For true forecasting, would need temporal lag features to capture delayed effects.

3Ô∏è‚É£ Geographic Variation Minimal in Cross-Sectional Data

Province/District combined only 0.3% importance

Why?

  • Case volume already captures urban (high) vs rural (low) patterns
  • Geographic differences embedded in health/social metrics
  • Cross-sectional model less sensitive to location

Implication: Time-series regional models might reveal location-specific patterns.

4Ô∏è‚É£ Cases 24h Dominance (89.9%) Is Both Strength & Weakness

Strength:

  • Model achieves 96.2% accuracy (excellent R¬≤)
  • Demonstrates ML methodology works correctly
  • Useful for nowcasting (predicting today from today's metrics)

Weakness:

  • Near-perfect correlation makes target identical to feature
  • True forecasting would need removal of this feature
  • Circular prediction (not useful for future forecasts)
5Ô∏è‚É£ Tree-Based Models Significantly Outperform Neural Networks

Random Forest & Gradient Boosting (R¬≤ > 0.93) vs Neural Network (R¬≤ = -0.36)

Reason: Small dataset (102 samples, 81 training) insufficient for deep learning. Neural networks require 500+ samples minimum.

Implication: For this dataset size, avoid complex models. Ensemble of tree-based models is optimal choice.

‚ñº ‚ö†Ô∏è Known Limitations

1. Cases 24h Feature Dominance (89.9%)

Model relies almost entirely on baseline feature. For true forecasting capability, this feature should be removed.

Expected Impact if Removed: R¬≤ would drop from 0.96 to ~0.65-0.75 (still acceptable, but reveals real predictive power)

2. Small Sample Size (102 total)

Limited data constrains model complexity and validation rigor.

Issues:

  • Cannot train deep learning models (NN failed)
  • Cross-validation limited (single 80/20 split used)
  • Generalization to new data uncertain

Ideal: 500+ samples for robust ML

3. No Temporal Ordering (Cross-Sectional Data)

Dataset lacks explicit dates (only years 2021, 2022, 2023). Cannot create lag features.

Missing: 7-day lag, 14-day lag, rolling averages, seasonal patterns

Impact: Model treats data as independent samples, not time-series. True forecasting requires temporal structure.

4. Limited Quarantine Signal (0.9%)

Social behavior measures surprisingly weak predictors. This could indicate:

  • Data quality issues
  • Measurement lags (quarantine is reactive)
  • Missing temporal relationships

5. Provincial Variance in Sindh (24.1% error)

Model struggles with Sindh predictions. Possible reasons:

  • High urban density (different dynamics)
  • Data quality variation
  • Insufficient Sindh samples for robust training
